A simple cloud-centric registration process

Resource discovery is simple. Register your thing by providing its URL e.g.: http://{ip of your Thing}. You can then configure your things, add resources to it, etc.
SicsthSense also supports a simple device discovery mechanism. To enable it, your Thing simply needs to provide a Json self-description hosted at /discover, following the following format (or any subset of it):
  {
    "uid": "uid",
    "resources": [
      "/sensors/temperature",
      "/sensors/energy"
    ]
  }

Registration through a simple RESTful API

We also provide a RESTful API for those who want to script registration operations:
POST /rest/things/register?url={thingUrl}
Register a Thing with and URL {thingUrl}. Returns the unique Thing identifier.
DELETE /rest/things/{thingId}
Delete Thing with identifier {thingId}
