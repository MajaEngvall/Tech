Getting started

The first step is to create an account. You can then add your data sources and start collecting their data. Sources provide a definition of how some data is accessed, and then streams of data points are populated from this source (e.g. poll or post).

When adding a source, SicsthSense can attempt to guess the correct way to parse the data by probing the data source URL. Json responses are parseable through simply specifying the path of any Json elements to save and plot. For example, the light and humidity of the following source would be stored by specifying the parsers "/resources/light" and "/resources/humidity". The values can be either sent as an integer or float. 

{
	"name": "New device",
	"sensors": {
		"light": 4.3,
		"temperature": 15,
		"humidity": 45
	}
}
Other data formats can be easily parsed by specifying a regular expression to capture relevant numbers from an external webpage. XPath support will be added shortly.

Polling data

The easiest way to get your sensor data into SicsthSense is to let us periodically poll your data source (e.g. device or webpage). This is solution is very flexible because it is entirely configured in the Cloud, through the Web interface. After creating your source, just discover or add new streams manually, with a corresponding polling interval. The data will soon appear in graphs, and be available through our RESTful API.

Posting data

Posting to SicsthSense is also useful, for delay-sensitive notification, or for data collection from devices with no public URL (e.g. a smartphone). To do so, just post at the following URL: 
  http://sense.sics.se/sources/[key]
Where key is a unique authorising key given by the system. Non-existing devices and resources you post to will be created on-the-fly.

Getting data

You can fetch data from SicsthSense's database through a GET at the following URI: 
  http://sense.sics.se/streams/[key]
Where key is a unique authorising key given by the system.

Interacting with devices

SicsthSense can act as a RESTful proxy towards any public device. This allows to share your devices without making their actual URI public. Just GET, POST, PUT or DELETE to: 
  http://sense.sics.se/proxy/[user]/[device]/[path]


Inspecting Users, Sources and Streams

You can access a subset of the data on users, devices and resources, in Json format. Simply get the following URIs: 
  GET        http://sense.sics.se/users/[user]
  GET        http://sense.sics.se/sources/[key]
  GET        http://sense.sics.se/streams/[key]
